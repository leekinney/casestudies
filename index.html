Challenges

The BAYADA website was in need of major updates when I was brought on to the team. The design was dated, had accessibility issues and was not mobile-friendly. The legacy code was almost unmanageable and the image and file structure was inconsistent and .

Improvements

My first step was to go through and clean up the HTML files, refactoring many of the pages to be semantic and compliant with basic accessibility standards.

  • Combining and minimizing files
  • Leveraging browser caching
  • Gzip compression
  • Image optimization
  • Removing IDs and renaming classes

Accessibility considerations

There were many issues with the current features of the site being inaccessible. Only the top level navigation items were accessible when tabbing through without a mouse and there was no way to skip over it to access important content. The font color palette provided by the brand guidelines did not provide enough contrast on a white background and was often used over other background colors, providing even less contrast.

Almost all of the images did not have alt text and the ones that did were done incorrectly, sometimes containing dashes or underscores or repeating the file name. Many of the hero images were created with a text overlay that did not have enough contrast and was done as an image. There were quite a few instances where text was done as an image instead of styling with CSS. This was troublesome as most examples were a call-to-action or button that would not be apparent to someone using the site with a screen reader. Using these images instead of CSS also added to the image bulk and slowed the pages down.

  • Skip-to navigation
  • Increased font-size and contrast
  • Image alt tags
  • Reworked drop down menu to be able to tab through
  • Removed image carousels throughout site

Results

  • Bounce rate improved by 28.73%
  • Organic traffic increase
  • Improved page speed by over 10%
